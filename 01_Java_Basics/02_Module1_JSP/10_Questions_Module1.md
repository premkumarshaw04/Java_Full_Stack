### between JDK, JRE and JVM, which one has physical existance.
```
JDK (Java Development Kit) → Has physical existence
It is a software package installed on your system containing the compiler (javac), JRE, libraries, and development tools.

JRE (Java Runtime Environment) → Has physical existence
It is also installed on your system and contains the JVM and runtime libraries needed to run Java programs.

JVM (Java Virtual Machine) → No separate physical existence
It is a virtual/abstract machine that exists as a process in memory when a Java program runs. It is implemented inside the JRE.

In one line:
JDK and JRE physically exist as installed software packages, while JVM is a virtual machine that runs inside the JRE.
```
### explain java program lifecycle.
```
Once we write a java program, it has to be saved in destination as .java Format.
This program will be verified with javac compiler.

If it finds any syntax mistake, then we get compile time error(BYTE CODE IS NOT GENERATD). If not javac generates Byte code.

Once the byte code is generated, It will be verified by JVM, If it finds any logical mistake, then we get exception(Program execution stops immedietly). If not JVM generates the machine code which will be processed by the machine and gives the final output.

```
### what is bytecode.
```
Bytecode is the platform-independent intermediate code generated by the Java compiler and stored in .class files, which is executed by the JVM to run a Java program.
```
### explain why java is considered as Platfrom Independent.
```
Java is considered platform independent because its source code is compiled into bytecode (.class file), not directly into machine code of a specific operating system.

The same bytecode can run on any system that has a compatible JVM (Java Virtual Machine) installed. The JVM converts the bytecode into machine code suitable for that particular platform.

For example, a Java program compiled on Windows can run on Linux or macOS without changing the code, as long as a JVM is available on those systems.

That is why Java follows the principle: “Write Once, Run Anywhere (WORA).

Important clarification (people often mix this up):
Java itself isn't platform independent — the JVM is platform dependent (you need the right JVM for the right OS). It's the bytecode that is platform independent, because it behaves the same regardless of where it runs, thanks to the JVM acting as a translator layer underneath.
```

```
Process of reassigning value to a variable is called as reinitialization.

```
### how to create constant values in java.
```
In Java, constant values are created using the final keyword. A variable declared as final can be assigned a value only once, and its value cannot be changed later.

example:
final double PI = 3.14159;
```
